Madcow is aimed at development and system-integration organisations that have a need to develop robust, automated, regression, test suites.

The Madcow philosophy lends itself to an agile, collaborative, development environment, where the development team are continuously delivering builds to the test team to test. If the test team use Madcow to assist them in covering integration, system and acceptance testing in an automated way, they can be freed up to conduct explorative, destructive, ad-hoc, useability, etc. testing – thus better utilising their time and expertise.

Madcow provides a Testing Philosophy, Process and Framework to an agile development organisation, which is comprised of:

  • Automated Developer Functional Testing – integrated into Development and Build activities
  • Automated System / Functional Scenario Testing – focusing on System Regression Testing
  • Acceptance Testing – simple way for Business users or SMEs (Subject Matter Experts) to specify acceptance criteria and actuate the tests
